┌──────────────────────────────────────────────────────────────────┐
│ IMPLEMENT vs USE                                                 │
├──────────────────────────────────────────────────────────────────┤
│                                                                  │
│ ❌ IMPLEMENT = You wrote the encryption code                     │
│    Example: You coded AES encryption from scratch               │
│                                                                  │
│ ✅ USE = You call Apple's APIs that have encryption              │
│    Example: You use URLSession which does HTTPS                 │
│                                                                  │
│ PitBox = USES Apple's encryption, does NOT IMPLEMENT            │
│                                                                  │
└──────────────────────────────────────────────────────────────────┘


╔══════════════════════════════════════════════════════════════════╗
║                      PITBOX TECHNICAL FACTS                      ║
╚══════════════════════════════════════════════════════════════════╝

What PitBox Uses:
  ✅ URLSession with HTTPS (Apple's networking stack)
  ✅ Supabase client (uses URLSession under the hood)
  ✅ iOS Keychain for secure storage (Apple's API)
  ✅ iOS file encryption (Apple's built-in)

What PitBox Does NOT Use:
  ❌ Custom encryption algorithms
  ❌ Third-party crypto libraries (OpenSSL, etc.)
  ❌ Manual AES/RSA/DES implementation
  ❌ Custom TLS implementation
  ❌ End-to-end encryption code
