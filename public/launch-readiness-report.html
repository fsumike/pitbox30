3. All App Icons Are Placeholder Files

Every icon file in /public/ is a 20-byte text file saying "[DUMMY FILE CONTENT]"

Files affected: android-icon-*.png, apple-icon-*.png, favicon-*.png, ms-icon-*.png

Impact: App stores will reject submission

How to Fix:
  1. Design a 1024x1024 master icon
  2. Use icon generator: https://appicon.co/ or https://icon.kitchen/
  3. Replace ALL placeholder .png files
  4. Copy to native platforms after generating them

4. iOS Splash Screens Are Placeholders

All 3 splash screen files are 20-byte placeholders in ios/App/App/Assets.xcassets/Splash.imageset/

Impact: App will show broken/blank splash screen

How to Fix: Create real 2732x2732 splash screen images

5. No Android Signing Configuration

No keystore file or signing configuration exists.

Impact: Cannot create release builds for Google Play Store

How to Fix: keytool -genkey -v -keystore pitbox-release.keystore -alias pitbox -keyalg RSA -keysize 2048 -validity 10000

Then configure in android/app/build.gradle

WARNING: Store keystore safely - if you lose it, you can NEVER update your app!

6. Too Many iOS Permissions Requested (53!)

Your capacitor.config.ts requests 53 different permissions, but you only use about 5.

Impact: Apple will reject your app for requesting unnecessary permissions

You only need: Camera, Photo Library, Location, Push Notifications

Remove: Microphone, Motion, Contacts, Calendar, Reminders, Bluetooth, FaceID, Siri, Speech Recognition, Apple Music, Health, HomeKit, NFC Reader, and ~20 more

How to Fix: Edit capacitor.config.ts and keep only the 5 permissions you actually use
