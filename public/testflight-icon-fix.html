üòä The Solution

iOS needs your icons in a specific location. We'll copy them there automatically!

1

Build iOS Project

First, make sure your iOS project exists:

npm run build

This generates the iOS folder if it doesn't exist yet.

2

Run Icon Fix Script

This automatically copies your icons to the right place:

npm run fix:ios-icons

The script will:

  • Copy all your icon files
  • Configure Contents.json
  • Set up all required sizes
3

Open in Xcode

Open your project to verify the icons are there:

npm run cap:open:ios

Then in Xcode:

  1. Click "App" in the left sidebar
  2. Click "App" under TARGETS
  3. Go to "General" tab
  4. Look for "App Icons and Launch Screen"
  5. You should see your icon now!
4

Archive & Upload

Create a new build for TestFlight:

  1. In Xcode: Product ‚Üí Archive
  2. Click "Distribute App"
  3. Choose "App Store Connect"
  4. Upload the build
üí° Pro Tip: Wait about 5-10 minutes for TestFlight to process your build.
5

Test It!

Install from TestFlight and check your home screen:

  • Open TestFlight app
  • Install the new build
  • Look at your home screen
  • Your PitBox icon should appear! üéâ

üìã Pre-Upload Checklist

  • Ran npm run build
  • Ran npm run fix:ios-icons
  • Opened Xcode and verified icons are visible
  • No yellow warnings in Xcode
  • Icon is 1024x1024 pixels
  • Icon has NO transparency (no alpha channel)
  • Icon has square corners (iOS rounds them)
  • Archived successfully in Xcode
  • Uploaded to App Store Connect
  • Tested on real device via TestFlight

You should see your icon in all these slots:

  • iPhone Spotlight (40pt) - 2x and 3x
  • iPad App (76pt) - 1x and 2x
  • App Store (1024pt) - 1x
‚úì No yellow triangles! Yellow warnings mean something is wrong. All slots should have icons.

üÜò Troubleshooting

Problem: Script says "iOS project not found"

Solution: Run npm run build first to generate the iOS project.

Problem: Icon appears in Xcode but not TestFlight

Solution: Make sure you:

  • Archived AFTER running the icon fix script
  • Uploaded the new build (check build number)
  • Waited 5-10 minutes for processing
  • Installed the NEW build number in TestFlight

Problem: Icon has white border or looks wrong

Your icon probably has transparency. iOS requires icons with NO alpha channel. Export your icon as RGB PNG without transparency.

Problem: Yellow warnings in Xcode

Some icon sizes are missing. Use an online tool to generate all sizes:

  • appicon.co
  • appicon.build
  • makeappicon.com
