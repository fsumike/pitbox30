üîß HOW TO USE THIS KEYSTORE
============================

When Building Release APK/AAB in Android Studio:
-------------------------------------------------

1. Menu: Build ‚Üí Generate Signed Bundle / APK...

2. Select: "Android App Bundle" (for Play Store)
   OR "APK" (for direct distribution)

3. Click: "Next"

4. Choose existing keystore:
   - Click "Choose existing..."
   - Browse to keystore file location (see above)
   - Enter keystore password (see above)
   - Select key alias (see above)
   - Enter key password (see above)

5. Click "Next"

6. Select build variant: "release"

7. Check signature versions: V1 and V2 (both)

8. Click "Finish"

9. Wait for build (5-15 minutes first time)

10. Look for success notification

11. Click "locate" to find output file


Output Locations:
-----------------
AAB:  android/app/build/outputs/bundle/release/app-release.aab
APK:  android/app/build/outputs/apk/release/app-release.apk

üÜò RECOVERY PLAN
=================

If Keystore File is Lost:
--------------------------
‚ùå You CANNOT recover it - there is no recovery process
‚ùå You CANNOT update existing Google Play Store app
‚ùå You must create NEW keystore
‚ùå You must publish as NEW app with NEW package name
‚ùå Users must uninstall old app and install new one
‚ùå You lose ALL reviews, ratings, and download counts

THIS IS WHY BACKUP IS ABSOLUTELY CRITICAL!


If Password is Forgotten:
--------------------------
‚ùå You CANNOT recover it - passwords cannot be reset
‚ùå Same consequences as losing keystore file
‚ùå You must create NEW keystore and NEW app

THIS IS WHY WRITING DOWN PASSWORDS IS ESSENTIAL!


If Keystore is Compromised:
----------------------------
1. Immediately create new keystore
2. Build new release with new keystore
3. Submit as new app to Play Store
4. Contact Google Play support to report issue
5. Notify users via all channels
6. Consider legal action if necessary
